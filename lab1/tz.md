# Техническое задание (кратко): социальная сеть

## 1. Цель проекта
Создать минимально жизнеспособную социальную сеть, где пользователь может:
- создавать, редактировать и удалять свои посты;
- оставлять комментарии под постами;
- ставить лайки на чужие посты;
- добавлять изображение к посту;
- устанавливать изображение профиля.

## 2. Роли и доступ
- **Гость**: просмотр публичной ленты и постов.
- **Пользователь** (авторизованный):
  - создавать/редактировать/удалять **свои** посты;
  - добавлять/удалять изображение к **своему** посту;
  - устанавливать/обновлять изображение профиля;
  - комментировать **любые** посты;
  - ставить лайк **чужим** постам;
  - снимать свой лайк.

## 3. Функциональные требования
### 3.1 Посты
- Создание поста: текст обязателен.
- Изображение к посту: опционально, не более 1 изображения.
- Редактирование: доступно только автору.
- Удаление: доступно только автору.
- Просмотр: доступен всем пользователям.

### 3.2 Комментарии
- Добавление комментария к любому посту.
- Просмотр комментариев по посту.
- Редактирование/удаление комментариев **не входит** в MVP.

### 3.3 Лайки
- Лайк можно поставить только на чужой пост.
- Повторный лайк одного и того же поста не допускается.
- Возможность снять лайк.
- Счетчик лайков отображается у поста.

### 3.4 Изображение профиля
- Пользователь может загрузить изображение профиля.
- Замена изображения профиля должна обновлять отображение в ленте и на странице поста.

## 4. Основные сущности (данные)
- **User**: id, name, email (или логин), avatarUrl, дата регистрации.
- **Post**: id, authorId, text, imageUrl, createdAt, updatedAt.
- **Comment**: id, postId, authorId, text, createdAt.
- **Like**: id, postId, userId, createdAt.
- **Media**: id, ownerType (post/user), ownerId, url, type (image), createdAt.

## 5. Экранные сценарии (минимум)
- Лента постов (список: текст, автор, аватар, время, количество лайков, количество комментариев, изображение поста).
- Страница поста (пост + комментарии + кнопка лайка).
- Создание/редактирование поста (форма + загрузка изображения).
- Профиль пользователя (просмотр и смена изображения профиля).

## 6. Нефункциональные требования
- Безопасность: авторизация обязательна для создания контента, редактирования и лайков.
- Локализация: интерфейс на русском языке.
- Хранение изображений: допускается использование внешнего хранилища (S3‑совместимого).

## 7. Ограничения и допущения
- Уведомления, подписки, личные сообщения — вне объема.
- Поиск и рекомендации — вне объема.

## 8. Критерии приемки
- Пользователь может создать, отредактировать и удалить **свой** пост.
- Пользователь может оставить комментарий под любым постом.
- Пользователь может поставить и снять лайк с **чужого** поста.
- Лента корректно отображает количество лайков и комментариев.
- Пользователь может добавить изображение к своему посту и заменить его.
- Пользователь может установить и заменить изображение профиля.
